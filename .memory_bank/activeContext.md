## Актуальная картина системы
- Приложение построено на Next.js App Router и в основном использует client components.
- Основные маршруты: `/`, `/rooms`, `/room/[id]`, `/rating`, `/profile`, `/register`, `/auth/signin`, `/auth/error`.
- Основные API-роуты: `api/auth/[...nextauth]`, `api/chat`, `api/recommendations`, `api/setup`.
- В `src/lib/supabase.ts` сосредоточен клиент Supabase и базовые API-обертки для комнат и профилей.
- При отсутствии конфигурации Supabase большая часть пользовательских сценариев деградирует до локального режима через `localStorage`.

## Открытые вопросы и риски
- Чат реализован через polling к API каждые 3 секунды, а не через realtime-подписки.
- Плеер хранит очередь и комментарии локально в браузере; серверная синхронизация треков ограничена.
