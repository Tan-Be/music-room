# Бриф Проекта - Music Room

## 1. Обзор
Music Room - веб-приложение для совместного прослушивания музыки в комнатах. Пользователь может авторизоваться через GitHub, создавать комнаты, приглашать участников, запускать музыку из YouTube, общаться в чате и работать как с Supabase, так и в локальном демо-режиме.

## 2. Цели проекта
- Дать быстрый способ создать музыкальную комнату и пригласить друзей.
- Сохранить базовую работоспособность даже без настроенного Supabase.
- Поддержать пользовательские сценарии профиля, рейтинга комнат и рекомендаций.

## 3. Границы проекта

### In-Scope
- Главная страница с навигацией и персональными рекомендациями.
- GitHub OAuth через NextAuth.
- Список комнат, создание и удаление комнат, переход в комнату.
- Страница комнаты с YouTube-плеером, QR-приглашением и чатом.
- Профиль пользователя с историей и избранным.
- Рейтинг комнат.
- Fallback в localStorage, если Supabase не настроен или недоступен.

### Out-of-Scope
- Собственный стриминг аудио и хранение музыкальных файлов.
- Полноценная real-time синхронизация плеера между участниками.
- Платежи, подписки и монетизация.
- Нативные мобильные приложения.

## 4. Критерии успеха
- Пользователь может пройти аутентификацию и открыть защищенные сценарии.
- Комнаты создаются и открываются в рабочем или демо-режиме.
- Внутри комнаты доступно добавление YouTube-треков и обмен сообщениями.
- Публичные страницы профиля и рейтинга доступны без критических ошибок.

## 5. Технические ограничения
- Основа приложения: Next.js App Router с клиентскими страницами.
- Данные и auth-интеграции завязаны на Supabase и NextAuth.
- Визуальный слой в основном построен на inline styles и локальном CSS.
- Пакетный менеджер проекта по правилам репозитория: bun.
