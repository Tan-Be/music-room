## Актуальная картина системы
- Приложение построено на Next.js App Router и использует маршруты `/`, `/rooms`, `/room/[id]`, `/rating`, `/profile`, `/register`, `/auth/signin`, `/auth/error`.
- Серверные route handlers находятся в `src/app/api/auth/[...nextauth]/route.ts`, `src/app/api/chat/route.ts`, `src/app/api/recommendations/route.ts`.
- Доступ к данным Supabase и обертки для работы с комнатами и профилями сосредоточены в `src/lib/supabase.ts`.
- При отсутствии рабочей конфигурации Supabase часть пользовательских сценариев продолжает работать через `localStorage`.
- В репозитории присутствует неотслеживаемый файл `nul`, который будет включен в коммит согласно прямому запросу пользователя на коммит и push всех файлов.

## Активные решения
- Канонический статус проекта считается только по разделу `## Project Deliverables` в `memory_bank/projectbrief.md`.
- Архитектурное описание хранится в `docs/README.md`; `memory_bank` фиксирует операционный контекст и подтвержденные наблюдения.
- Для публичных страниц и ключевых компонентов используются записи в `memory_bank/ui_extension/`.

## Открытые вопросы и риски
- В `package.json` указан `packageManager: pnpm`, хотя репозиторное правило требует использовать `bun`.
- Чат по-прежнему работает через polling, а не через realtime-подписки.
- Очередь треков и комментарии плеера остаются в локальном хранилище браузера и не синхронизируются сервером в полном объеме.
